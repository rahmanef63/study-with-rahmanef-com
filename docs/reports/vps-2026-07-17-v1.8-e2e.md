## Fix mekanis (§4 vps exception — file test, ZERO backend/data risk, flag alpha)

**Gejala:** run pertama spec 8 **GAGAL** — bukan di assertion UI (teks
"Sertifikat tidak ditemukan" TERLIHAT, no-crash lolos) tapi di filter
console-error di akhir spec.

**Akar masalah (bukan bug prod):** verifikasi langsung ke prod (read-only)
```
npx convex run --prod features/profiles/public:publicGetCertificate '{"completionId":"e2e-bogus-completion-id"}'
→ Uncaught ConvexError: {"code":"NOT_FOUND","message":"Sertifikat tidak ditemukan"}
     at certificateNotFound (convex/features/profiles/public.ts:116)
```
Prod **BENAR**: `publicGetCertificate` melempar `ConvexError` NOT_FOUND uniform
(no-oracle §6), UI render copy not-found SSOT. Yang salah cuma **regex allowlist
spec**: filter lama (`635b904`) mencari teks `NOT_FOUND` / `Sertifikat tidak
ditemukan` di string console. Di **bundle browser PRODUKSI**, pesan yang dicetak
`console.error` adalah wrapper ter-scrub `... [CONVEX Q(...publicGetCertificate)]
[Request ID: …] Server Error` — payload `{code:"NOT_FOUND"}` menumpang di
`.data`, **bukan** di string yang dicetak. (CLI/Node mencetak payload; browser
prod tidak.) → regex tak pernah match → error by-design dianggap "unexpected".

**Fix (1 baris, `e2e/smoke.anon.spec.ts`):** anchor allowlist ke **path query**:
```diff
- (e) => !/NOT_FOUND|Sertifikat tidak ditemukan/.test(e),
+ (e) => !/publicGetCertificate|NOT_FOUND|Sertifikat tidak ditemukan/.test(e),
```
Paling ketat: **hanya** error by-design dari query sertifikat itu yang disaring;
error console fungsi lain / crash nyata TETAP menggagalkan spec, dan assertion
teks not-found di atasnya sudah menjaga crash sertifikat yang sebenarnya. Prod
tidak disentuh; tidak ada deploy.

**Proposal (storage-state privat, TIDAK dieksekusi tanpa persetujuan owner):**
Kalau owner ingin suite authed jalan periodik lawan **staging** (prod ditolak

`~/e2e-auth/user.json`, `chmod 600`) dan suite dijalankan dengan file itu

yang merekam, rotasi saat sesi kedaluwarsa). vps tidak menjalankan tanpa "ya".
